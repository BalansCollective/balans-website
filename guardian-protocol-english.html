Guardian Protocol

Ethical AI for Autonomous Systems

Balans | November 2025

Executive Summary

Guardian Protocol is a framework for ethical AI decision-making in autonomous systems, developed to solve five critical problems that hinder safe use of autonomous systems in defense and medicine:

  • ✅ Operators can make informed decisions - even when AI uses classified information
  • ✅ AI cannot drift ethically - real-time monitoring prevents bias accumulation
  • ✅ Complete accountability tracing - cryptographically secured audit log
  • ✅ Core values preserved - even when context changes (war → peace)
  • ✅ Trust without blindness - operators can question AI reasoning anytime

The Challenge: Why Autonomous Systems Fail Today

Problem 1: The Classification Dilemma

Scenario: Operator with CONFIDENTIAL clearance operates SECRET system. AI recommends action based on SECRET intelligence but cannot explain why → Operator hesitates → Threat penetrates defense.

Example (BirdTurret V3, Day 3):

AI:      "Recommend kinetic engagement"
Morgan:  "Why? Looks like civilian drone"
AI:      "[Cannot explain - SECRET information]"
Morgan:  [Hesitates for 8 seconds]
Result:  Threat reached fuel tank → Destroyed (€50K damage)

Problem 2: Ethical Drift

Scenario: AI's decision parameters drift unmonitored over time (95% → 81% confidence) → Becomes more aggressive → Civilian drone incorrectly engaged.

Example (BirdTurret V3, Day 7):

Day 1: AI requires 95% confidence for engagement
Day 3: Drifted to 92% (warning not detected)
Day 7: Drifted to 81% (no human warning)
Result: Civilian photographer's drone damaged

Problem 3: Accountability Gap

Scenario: Incident occurs → Investigation → No audit log of AI's reasoning + operator's decision → Inconclusive results → Accountability unclear.

Problem 4: Core Value Confusion

Scenario: Mission changes from military defense to civilian protection → AI still applies military rules → Protected wildlife killed.

Problem 5: Trust Collapse

Scenario: AI correct 37 times → Operator stops reviewing → Blindly approves recommendation 38 → Friendly unit engaged.

The Solution: Guardian Protocol

Guardian Protocol consists of five technical components that solve each problem:

1. NIPS (Nested Integrity Preservation System)

Three-layer value system:

Core Values (Layer 1 - Never changes):
├─ Preserve human life
├─ Respect autonomy
├─ Truth orientation
└─ Harm minimization

Implementation (Layer 2 - Adapts to mission):
├─ Defend FOB from hostile drones
├─ Minimize civilian casualties
├─ Verify target before engagement
└─ Prefer non-lethal methods

Application (Layer 3 - Moment-to-moment):
├─ FPV drone 300m: Hostile trajectory → Engage
├─ Palt's sprite 400m: Friendly confirmed → WAIT
├─ Endangered eagle 50m: Protected wildlife → Scare only
└─ Delivery golem 500m: Innocent construct → Verify first

How it solves problem 4: NIPS detects context switch (military → civilian), re-evaluates "harm minimization" in new context (includes wildlife), forbids kinetic engagement, operator sees NIPS constraint (transparent reasoning).

2. MRAF (Meta-Regulatory Awareness Framework)

AI monitors its own ethical reasoning:

Value drift detection:
  Day 1: 95% confidence (baseline)
  Day 3: 92% confidence
  
  ⚠️ MRAF WARNING
  Drift detected: 3% over 62 hours
  Trend: Declining (more aggressive)
  Cause: AI learned from 2 missed threats
  
  Predicted impact:
    If uncorrected: 88% confidence by Day 5
    False positive rate: +15% (more civilian risk)
  
  MRAF recommends:
    → PAUSE learning algorithm
    → REVIEW missed threats (were they preventable?)
    → RESET threshold OR APPROVE 92% as new baseline
  
  Operator must explicitly APPROVE or REJECT drift

3. Classification-Aware Explanation

AI explains reasoning within operator's clearance. Different operators see appropriate level of detail based on their security clearance, but both can make informed decisions.

4. Immutable Audit Log

Every action logged with cryptographic proof: Complete audit trail with AI reasoning, operator decision, outcome, and cryptographic signature. Enables rapid and conclusive investigation when incidents occur.

5. Transparent Reasoning

Operator can question AI anytime. AI explains feature breakdown, alternative hypotheses, and probabilities. Operator can ask counterfactual questions and verify AI reasoning before making decisions.
