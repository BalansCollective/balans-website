Forge Protocol

Klassificeringsmedveten AI-Utveckling

Balans | November 2025

Sammanfattning

Forge Protocol är ett ramverk för att bygga AI-system över säkerhetsgränser, utvecklat för att lösa fem kritiska problem som hindrar säker AI-utveckling inom försvar:

  • ✅ Utvecklare med olika behörigheter kan samarbeta - utan att läcka hemligheter
  • ✅ Kod kan byggas och testas - även när specifikationer är hemliga
  • ✅ Efterlevnad är bevisbar - automatisk validering mot standarder
  • ✅ Revision är komplett - full provenance från krav till deployment
  • ✅ Innovation accelererad - ingen 6-månaders SÄPO-väntan för varje startup

Sister Protocol: Guardian Protocol hanterar operativ säkerhet (deployed AI). Forge Protocol hanterar utvecklingstidssäkerhet (building AI).

Utmaningen: Varför Klassificerad AI-Utveckling Misslyckas Idag

Problem 1: Behörighetsparadoxen

Scenario: Senior utvecklare (CONFIDENTIAL behörighet) får krav: "Implementera SECRET algoritm från specifikation doc-00423." Kan inte läsa spec → Kan inte implementera korrekt → Systemet är osäkert.

Problem 2: Data Diode-gapet

Scenario: AI tränas på SECRET data i "Red Cell" (säker miljö). Måste testa performance → Data UT ur Red Cell → Säkerhetsbrott.

Problem 3: Standardefterlevnadsgap

Scenario: FMV kräver bevis för EU AI Act-efterlevnad. Utvecklingsteam har dokumentation i Word/PDF → FMV-revisor måste manuellt korrelera krav → 4 timmars granskning per incident.

Problem 4: Provenance-förlust

Scenario: Säkerhetsincident → Utredning behöver veta "Vem ändrade denna parameter? Varför? Godkändes det?" → Git-loggen saknar kontext → Okonklusiv utredning.

Problem 5: Startup-innovationshinder

Scenario: NATO DIANA startup vill bygga AI-lösning för försvar → Behöver SÄPO-godkännande (€50K, 6 månader) för klassificerad utveckling → Kan inte iterera snabbt → Förlorar kontrakt till etablerad aktör.

Lösningen: Forge Protocol

Forge Protocol består av fem tekniska komponenter som löser varje utvecklingstidsproblem:

1. Red Cell Architecture (Klassificerad Utvecklingsmiljö)

Säker "sandbox" för klassificerad AI-utveckling. SECRET data lämnar aldrig Red Cell (data diode IN endast), test metrics kan exporteras (inga hemligheter), utvecklare får feedback för iteration utan att se hemlig data.

2. Classification-Aware Development Flows

Utvecklare med olika behörigheter samarbetar utan läckage. CONFIDENTIAL utvecklare ser filtrerade specifikationer med SECRET sektioner borttagna. SECRET spec-författare arbetar i Red Cell med full detalj. Compliance Plugin filtrerar automatiskt innehåll baserat på behörighet.

3. Standards-as-Graphs (Frågebara Efterlevnadskrav)

EU AI Act-krav lagras som frågebara grafer (Neo4j). Istället för manuell PDF-granskning kan FMV-revisorer köra Cypher-queries för omedelbar efterlevnadsvalidering. Automatiska deployment gates förhindrar deployment utan komplett bevis.

4. Git Provenance Tracking

Varje kodändring spårad med AI-contribution + human-review. MCP Git Tool fångar: vem ändrade vad, varför, konfidensnivå, AI vs. mänsklig contribution procent, granskningscykler, efterlevnadslänkar. Möjliggör snabb incident-utredning med fullständig kontext.

5. Zenoh Data Diode Enforcement

Information flödar IN till Red Cell, aldrig UT (utom metrics). Compliance Plugin (Rust WASM) interceptar ALLA Zenoh-operationer. Whitelist-approach: endast precision, recall, F1, latency, throughput metrics tillåtna ut. Images, RF-profiler, koordinater blockeras automatiskt.

Värde för Svensk Industri

För Försvarsindustri

Problem: Utveckla AI-system över säkerhetsgränser kräver dyra, separata team för varje behörighetsnivå → Långsam iteration, svårt samarbeta.

Forge Protocol-lösning: Senior devs (CONFIDENTIAL) implementerar från filtrerade specs, SECRET spec writers skriver full detalj i Red Cell, automatisk efterlevnadsvalidering.

Värde: Snabbare marknadsintroduktion, lägre utvecklingskostnader, enklare FMV-certifiering.

För Mindre Försvarsaktörer

Problem: Behöver SÄPO-godkännande (€50K, 6 månader) för klassificerad AI-utveckling → Kan inte konkurrera med etablerade aktörer.

Forge Protocol-lösning: Dela Red Cell-miljö (Classified Development as a Service), ingen egen SÄPO-ansökan, pay-per-use modell (€5K/månad vs. €50K capex).

Värde: Snabbare innovation (iteration från dag 1), lägre entry cost, FMV-access.

För Försvarsforskning

Problem: Forskning på hemliga hot kräver säker miljö → forskare kan inte samarbeta med externa akademiker.

Forge Protocol-lösning: Skriv SECRET specs i Red Cell, externa forskare implementerar från CONFIDENTIAL specs, Standards-as-Graphs möjliggör publikation (delar resultat, inte hemligheter).

Värde: Bredare samarbete, snabbare publikation, enklare EU-finansiering.
